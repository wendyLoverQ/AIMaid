## Core 请求处理失败 - Gemini ApiKey 修复

### 问题
Core 请求处理失败，Gemini 模型的 ApiKey 未被正确传递，导致所有依赖 Gemini 的 LLM 调用失败。

### 根因分析
1. **数据库配置**：Gemini 的 `user_config:App:Model:Gemini:ApiKey` 在 AppSettings 表中已正确存储
2. **代码缺陷**：`LegacyRelationalDocumentStore.GetModelConfigurationAsync` 的 `ModelFields` 不包含 `"ApiKey"`，且硬编码 `["ApiKey"] = ""`，导致从 domain store 读取的配置覆盖了 settings 中正确配置的 ApiKey
3. **反映路径**：`ExtendedDomainApplicationService.HandleAsync(ListModelConfigurationsQuery)` 先正确构建配置，然后被 domain store 的空 ApiKey 覆盖

### 修复内容
1. **`LegacyRelationalDocumentStore.cs`**：
   - `ModelFields` 添加 `"ApiKey"` 
   - `GetModelConfigurationAsync` 改为从 settings 读取 ApiKey：`["ApiKey"] = fields.GetValueOrDefault("ApiKey", "")`
2. **`HttpIntegrationClients.cs`**：
   - `DescribeErrorBody` 兼容 Gemini 返回的 JSON 数组错误响应 `[{error:{...}}]`（非 Object 类型）
3. **数据库**：Gemini Endpoint 配置为 `https://generativelanguage.googleapis.com/v1beta/openai/chat/completions`
4. **WAL**：清理了 203MB 的 WAL 文件

### 测试结果
- qwen3-14b (本地模型)：✅ 正常
- Gemini (远程 API)：✅ 正常，成功返回 "Hello."

## JSON 双重序列化全面修复

### 检查范围
搜索了 `src/` 下所有 `.cs` 文件，检查了 18 种 JSON 相关模式。

### 发现的 Bug
1. **`CoreProtocolHost.cs:788`** — `argsJson` 双重序列化（将 JSON 字符串嵌入 Dictionary 导致二次转义，客户端需 `JSON.parse(error.details.argsJson)` 才能获取）
2. **`HttpIntegrationClients.cs:65/90`** — 同一 payload 被 Serialize 两次（性能浪费）
3. **`HttpIntegrationClients.cs:174`** — `Regex.Unescape` 误用于 JSON 字符串（语义错误）
4. **`CoreProtocolHost.cs`** — 17 处 `new JsonSerializerOptions(JsonSerializerDefaults.Web)` 重复实例化

### 已确认安全的模式
- `capabilitiesJson` / `recentMessagesJson` 在 template values 中用作字符串替换，不经过二次序列化
- `OutputSchemaJson` / `templateCardJson` 硬编码 JSON 字符串在 C# 中是正常转义，单次序列化后正确
- `ArgsSchemaJson` 的 `Deserialize<JsonElement>` 后立即 `Serialize` 是正确的内联模式
- 所有 `ConfigJson` / `ArgsSchemaJson` 在消费者端正确使用 `JsonDocument.Parse`

### 修复
| 文件 | 修改 |
|------|------|
| `CoreProtocolHost.cs` | argsJson → `JsonSerializer.Deserialize<JsonElement>`；17 处 JSOS → `static JsonOptions` |
| `HttpIntegrationClients.cs` | 清除重复 Serialize；移除 `Regex.Unescape` |
| `ProtocolWriter.cs` | 使用 `JsonConfig.Web` |
| `TemplateCardApplicationService.cs` | 使用 `JsonConfig.Persistence` |
| `BinanceMarketApplicationService.cs` | 使用 `JsonConfig.Web` |
| `JsonConfig.cs` (新增) | Web/Audit/Persistence/Readable 四个集中配置 |

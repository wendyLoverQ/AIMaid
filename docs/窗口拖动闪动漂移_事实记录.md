# AIMaid 窗口拖动闪动与漂移问题事实记录

## 1. 问题描述

- 用户反馈：拖动 AIMaid 窗口时，屏幕出现闪动，窗口出现漂移。
- 用户反馈：该现象存在于所有 AIMaid 窗口。
- 本文只记录当前代码和运行环境中可以直接确认的事实，不给出原因判断。

## 2. 当前桌面宿主

- 桌面宿主位于 `apps/desktop`。
- `apps/desktop/package.json` 中声明的 Electron 版本为 `43.1.1`。
- Electron 主进程入口为 `apps/desktop/src/main/main.ts`。
- 所有 `BrowserWindow` 均通过 `apps/desktop/src/main/windows/window-factory.ts` 中的 `WindowFactory.create()` 创建。

## 3. 公共窗口创建代码

文件：`apps/desktop/src/main/windows/window-factory.ts`

```ts
create(definition: WindowDefinition): BrowserWindow {
  const window = new BrowserWindow({
    ...definition.options,
    title: `AIMaid - ${definition.id}`,
    webPreferences: {
      preload: this.preloadPath,
      contextIsolation: true,
      nodeIntegration: false,
      sandbox: true,
      webSecurity: true,
      allowRunningInsecureContent: false,
      experimentalFeatures: false,
      webviewTag: definition.id === 'douyin-login',
      spellcheck: false,
      backgroundThrottling: definition.id !== 'pet',
      additionalArguments: [
        `--aimaid-window=${definition.id}`,
        `--aimaid-version=${app.getVersion()}`
      ]
    }
  })
}
```

可直接确认：

- `WindowFactory.create()` 先展开 `definition.options`，再设置公共标题和 `webPreferences`。
- 该公共创建代码没有设置 `parent` 或 `owner`。
- 该公共创建代码没有注册 `move`、`resize`、`will-move` 或 `will-resize` 事件。
- 该公共创建代码没有调用 `setBounds()`、`setPosition()` 或 `setSize()`。
- 该公共创建代码没有为普通窗口统一设置 `backgroundColor`。
- `pet` 的 `backgroundThrottling` 为 `false`，其他窗口为 `true`。

## 4. 普通窗口基础定义

文件：`apps/desktop/src/main/windows/window-registry.ts`

普通模块窗口通过以下函数定义：

```ts
function moduleWindow(
  id: WindowKind,
  width: number,
  height: number,
  minWidth: number,
  minHeight: number
): WindowDefinition {
  return {
    id,
    route: id,
    closeBehavior: 'hide',
    options: {
      width,
      height,
      minWidth,
      minHeight,
      frame: false,
      resizable: true,
      show: false
    }
  }
}
```

使用 `moduleWindow()` 的窗口包括：

- `status`
- `appearance`
- `bitcoin`
- `video`
- `remote-video`
- `reminders`
- `notebook`
- `vault`
- `scripts`
- `voice-conversation`
- `characters`
- `crypto-events`
- `crypto-provider`
- `crypto-chart`
- `video-player`
- `video-subtitles`
- `remote-site-config`
- `template-card`
- `character-editor`
- `douyin-login`
- `ui-showcase`

可直接确认：

- 上述普通模块窗口均设置 `frame: false`。
- 上述普通模块窗口均设置 `resizable: true`。
- 上述普通模块窗口均设置 `show: false`。
- 上述普通模块窗口没有设置 `transparent`。
- 上述普通模块窗口没有设置 `parent` 或 `owner`。

`main` 和 `settings` 没有调用 `moduleWindow()`，但它们各自的 `options` 同样包含：

```ts
frame: false,
resizable: true,
show: false
```

## 7. 主进程窗口事件

文件：`apps/desktop/src/main/windows/window-manager.ts`

普通窗口创建后注册的公共事件包括：

```ts
window.on('close', ...)
window.on('closed', ...)
```

此外：

```ts
if (kind === 'chat') window.on('blur', () => window.hide())
if (kind === 'tray-menu') window.on('blur', () => window.hide())
```

可直接确认：

- 普通窗口没有公共 `move` 事件处理器。
- 普通窗口没有公共 `resize` 事件处理器。
- 普通窗口关闭时，`closeBehavior: 'hide'` 的窗口会阻止销毁并执行 `hide()`。
- `chat` 和 `tray-menu` 失去焦点时会执行 `hide()`。

### 8.2 音乐可视化窗口

文件：`apps/desktop/src/main/windows/window-manager.ts`

```ts
private attachMusicVisualizer(window: BrowserWindow): void {
  const pet = this.get('pet')
  if (pet !== undefined) {
    const sync = (): void => {
      if (!window.isDestroyed() && !pet.isDestroyed()) {
        window.setBounds(pet.getBounds(), false)
      }
    }
    sync()
    pet.on('move', sync)
    pet.on('resize', sync)
  }
}
```

可直接确认：

- `music-visualizer` 会复制 `pet` 的窗口 bounds。
- `music-visualizer` 监听 `pet` 的 `move` 和 `resize`。
- 该同步代码只在打开 `music-visualizer` 时安装。

## 9. 桌宠窗口输入模式

文件：`apps/desktop/src/main/windows/pet-window-manager.ts`

```ts
window.setIgnoreMouseEvents(
  ignore,
  ignore ? { forward: true } : undefined
)
```

文件：`apps/desktop/src/renderer/shared/pet-item-interaction-controller.ts`

- 桌宠 renderer 监听 `mousemove`、`pointerdown`、`pointermove`、`pointerup`、`pointercancel`、`wheel`、`blur` 和 `resize`。
- 桌宠元素拖动通过修改元素的 CSS `transform` 完成。
- 桌宠元素拖动代码没有调用普通业务窗口的移动 API。

## 10. 窗口之间的关系

从当前窗口定义和创建代码可直接确认：

- 普通业务窗口之间没有设置 `parent` 或 `owner`。
- 普通业务窗口之间没有位置同步代码。
- 普通业务窗口与 `pet` 之间没有位置同步代码。
- `music-visualizer` 与 `pet` 之间存在明确的 bounds 同步代码。
- `pet`、`chat` 和 `tray-menu` 通过 `alwaysOnTop: true` 设置窗口层级。

## 并行规则

1. 三个任务都必须先读根目录 `AGENTS.md` 和自己的交接文件。
2. 保留工作区所有未提交修改；禁止 `reset`、`checkout`、清理工作树或整文件格式化。
3. 各任务只改自己拥有的前端目录。共享 C# 路由只允许窄范围追加，优先新建独立 service/contract 文件。
4. 不运行 `npm run pack`；并行期间也不运行 `npm run pack:local`，避免杀进程和覆盖发布目录。
5. 每个任务完成时只运行：根目录 `dotnet build AIMaid.sln -c Release`，以及 `apps\desktop` 下 `npm run typecheck`。
6. 最终合并审计、进程清理和轻量发布由当前任务统一执行。
7. 用户自行操作 UI 验收；不得使用鼠标自动化代测。

## 完成定义

“抄完”必须同时满足入口、布局、文案、状态、条件显隐、确认流程、持久化和真实业务动作与老项目一致。只有静态页面、假状态、localStorage 临时通信或“Core 路由待接”提示不算完成。

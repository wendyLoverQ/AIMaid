# 后续业务模块纵向迁移模板

1. 在编码前记录模块名、旧入口、现有 Core 服务、数据表、Electron 入口和明确不做的范围。
2. 先定义领域命名的 Query、Command、Event（仅在真实需要时）和错误码，再同步 C# DTO、JSON 契约、TypeScript 类型与运行时校验。
3. 所有业务规则和最终校验放在 Core；Main 只做授权、路由、超时、取消和错误映射；Preload 只暴露模块专用 API。
4. 首次挂载和 Core 重连必须 Query。副作用 Command 必须说明幂等键、自动重试策略、结果不确定状态和并发冲突规则。
5. Renderer 分离 API、状态协调和页面；至少覆盖 Loading、Error、Offline，以及该模块真实需要的提交、冲突、空态或进度。
6. 测试覆盖 Core 正常/非法/持久化路径、契约校验、权限边界、Renderer 状态和真实 Core 集成；最后在生产包中完成真实查询与写入冒烟。
7. 交付文档记录最终链路、数据库策略、响应/Event 顺序、错误码和可复现的运行证据。
